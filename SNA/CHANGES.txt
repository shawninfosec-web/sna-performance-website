DESIGN DIRECTION
---------------------------------------------------
The site was redesigned to feel closer to Porsche and
Apple — restrained, editorial, and premium. The old
design was loud and somewhat generic; the new design
leans on typography, spacing, and materials rather
than color and emoji to do the heavy lifting.

KEY DESIGN CHANGES
---------------------------------------------------

1. TYPOGRAPHY
   • Dropped Bebas Neue (too rugged for a premium feel).
   • Primary typeface: Inter (weights 200–800) — clean,
     modern, and used at large sizes with tight negative
     letter-spacing for display headlines.
   • Accent typeface: Instrument Serif (italic only) —
     used for select words inside headlines (e.g.
     "Precision.", "your vision") for an editorial
     feel, similar to how Porsche and Apple blend
     sans + serif flourishes.
   • Body text is lighter (weight 300), more generous
     line-height, and has subtle negative tracking.

2. COLOR PALETTE
   • Base: true black (#000) with dark surface tiers
     (#0a0a0a, #111, #171717).
   • Text: a warm off-white (#f5f5f7) — Apple's text
     tone, easier on the eyes than pure white.
   • Borders: barely-visible rgba whites (8%, 14%)
     instead of hard grey lines.
   • Accent red changed from #dc2626 (hot red) to
     #e10600 (Porsche-inspired red) and is now used
     sparingly — for small accents, not as the primary
     visual weight.

3. LAYOUT & SPACING
   • Max width increased to 1320px for cinematic feel.
   • Section padding is now fluid (80–140px vertical).
   • Generous whitespace between headings, sub-copy,
     and grids.
   • Cards are unified into a single bordered grid
     (tiled 1-pixel seams) rather than floating cards
     with shadows — a more architectural feel.

4. HERO
   • Full-viewport hero with massive display type
     (up to ~150px on desktop).
   • Radial gradients replace the bright red glow,
     giving a moody, premium atmosphere.
   • Stat row now sits below a hairline rule,
     minimalist instead of boxed badges.

5. NAVIGATION
   • Glass / blur effect with subtle border.
   • CTA is now a pill-shaped white button (Apple-style)
     instead of a loud red rectangle.
   • Active nav link is indicated with a hairline
     underline.

6. ICONS
   • Replaced dozens of emoji icons (🏎️🔧🔊💡 etc.)
     with clean, minimal stroke-based SVG icons
     across index.html and services.html cards.
   • Contact page icons (📍📞✉️🕐) also converted
     to SVGs.
   • Instagram icon in footer is now an SVG.
   • Kept a handful of emojis where they clearly read
     as playful (IG avatar initials show as text now).

7. BUTTONS
   • Pill-shaped (999px radius) with refined spacing.
   • Three variants: primary (white on black),
     secondary (outlined glass), accent (red).

8. CARDS
   • Service cards redesigned — small SVG icon, refined
     type hierarchy, hairline separator before the
     feature list, and each list bullet uses a small
     red tick instead of a triangle.

9. FORMS
   • Inputs have deeper black backgrounds with subtle
     borders, thinner labels, and a gentle focus state.
   • Checkboxes are now styled as individual pill cards.
   • File upload area uses a soft dashed border and SVG
     camera icon.

10. FOOTER
    • Clearer two-part layout with hairline dividers.
    • Typography is now tighter and more refined.
    • Copyright year updated to 2026.


FORM BACKEND: WEB3FORMS
---------------------------------------------------
Switched off Formspree and onto Web3Forms — free,
unlimited submissions, no plan required. This pairs
cleanly with GitHub Pages hosting and lets Zapier's
free Email Parser trigger handle downstream routing
(SMS, Google Sheets, etc.) without needing the paid
$19.99/mo Zapier Starter plan.

Files updated:
  • quote.html   → action="https://api.web3forms.com/submit"
  • contact.html → action="https://api.web3forms.com/submit"

Both forms:
  • POST to Web3Forms with a hidden access_key field
  • Use a `botcheck` checkbox honeypot (Web3Forms'
    spam field, replacing Formspree's `_gotcha`)
  • Use a fetch-based submit handler so users see an
    in-page "Thanks" message instead of being redirected
    to Web3Forms' raw JSON response
  • Display a styled error block if submission fails
  • Preserve existing validation (required fields,
    agreement checkbox on quote.html)

Email subject lines:
  • quote.html   → "New Quote Request — SNA Performance"
                   (set via hidden subject field)
  • contact.html → whatever the user picks in the
                   "Subject" dropdown (Web3Forms auto-
                   uses the `subject` field as subject)

ZAPIER EMAIL PARSER — NEXT STEP (FREE PATH)
---------------------------------------------------
To route submissions into SMS, Google Sheets, etc.
without Zapier's paid webhook trigger:
  1. In Zapier, create a new Zap.
  2. Trigger: "Email Parser by Zapier → New Email".
     Zapier provides a parser inbox address
     (e.g. abc123@robot.zapier.com).
  3. In Gmail, add a filter that forwards every
     email from Web3Forms to that parser address.
  4. Submit a test through the site, open the email
     in the parser, and highlight each field (name,
     email, phone, message) so Zapier learns the
     template.
  5. Action steps: "Google Sheets → Create Row",
     or "Email → Send via Gmail" to your phone's
     email-to-SMS gateway (e.g. 5555551234@vtext.com
     for Verizon), or anything else.

This path works on Zapier's FREE plan.
